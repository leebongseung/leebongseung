## How I work

> **Understand the flow first. Change it safely. Verify it in operation.**

- 구현 전에 기존 흐름과 변경 영향을 먼저 확인합니다.
- 권한, 예외 처리, 데이터 일관성, 운영 관점의 검증을 함께 고려합니다.
- 재현 가능한 문서와 작은 단위의 변경을 선호합니다.
